## 将来の拡張 / 技術メモ
### 改稿検知ロジック案
- **小説家になろう**: 投稿日時文字列に「（改）」が含まれる場合、または投稿日時自体がローカル保存版と異なる場合に再ダウンロードを行う。
- **カクヨム**: エピソード更新日時（`publishedAt` または `lastEpisodePublishedAt`）の変化をトリガーにする。JSONデータ内に詳細なタイムスタンプが含まれているため、これをローカルの `info.json` と比較することで検知可能。
- 実装時は、`checkEpisodeExists` 関数を拡張し、単なるファイル存在確認だけでなく、メタデータのタイムスタンプ比較を行うように変更する。

### マルチユーザー対応 / アクセス制御
- **ユーザーごとのリスト表示分離**:
    - 現状は全ユーザーのリストが見えてしまう（またはタブ切り替えのみ）が、これを「指定したID（ログインID等）のリストしか表示しない」形式に対応させる。
    - URLパラメータや簡易ログインセッションを用いて、表示するリストをフィルタリングする仕組みを導入する。
    - 想定: 家族や友人とサーバーを共有する場合に、各自の「お気に入り」が混ざらないようにするため。
