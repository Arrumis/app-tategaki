
# **tategaki 仕様書（お気に入り小説を紙の本のように読むサイト）**

## 1. 目的
- 小説家になろう・カクヨムの小説を**縦書き**で読むことができるサイトを提供
- お気に入りの小説の**更新を追跡・管理**し、削除されてもサーバーにデータが残る仕組みを実装

## 2. ターゲットユーザー
- 本人（公開を検討）

## 3. 主な機能
| 機能 | 詳細 |
|------|------|
| **縦書き表示** | 元のサイトの小説を縦書きに変換（例: 横書き→縦書きのCSS変更） |
| **お気に入り登録** | 小説のURLを登録 → バックグラウンドで差分取得 |
| **更新差分の自動ダウンロード** | 元のサイトの更新を監視 → 既存データと比較して差分をDL |
| **データの持続性** | 元のサイトが削除されても、サーバーにデータが残る（削除防止のための仕組み） |

## 4. UIデザイン
| 要件 | 詳細 |
|------|------|
| **カラーテンプレート** | 本を読んでいるように感じる色（例: シャンプーの青、紙の白、茶色のフォント） |
| **ボタン配置** | 8つのボタンを以下に配置（画面の中央）<br>1. 次話 | 2. 次ページ | 3. 目次 | 4. 前ページ | 5. お気に入り | 6. 設定 | 7. ホーム |
| **デザインの特徴** | バックグラウンドの色が紙の質感を模している（例: シンプルなグレー・クリーム） |

## 5. 技術仕様
### (1) データ保存
- **データ構造**: JSONファイル（小説のURL、更新日時、差分データを含む）
- **保存場所**: サーバーの`/data`フォルダ（例: `novel_123.json`）
- **削除防止**: 元のサイトが削除されても、JSONファイルを`/backup`フォルダに移動（削除されてもサーバーに残る）

### (2) 更新監視と差分取得
| 機能 | 実装方法 |
|------|----------|
| **定期的なチェック** | Node.jsの`cron`で3時間おきに実行 |
| **差分取得** | 1. 元のサイトの最新HTMLを取得<br>2. 既存JSONと比較 → 新規追加/削除の差分を抽出 |
| **バックグラウンド処理** | Workerスクリプトで実行（ブラウザの背景タスクで可能） |

### (3) サイト構造変更時の対応
- **対応策**: 
  - 元のサイトのHTML構造が変更された場合、**バージョン管理**を実装（例: `novel_123_v1.json` → `novel_123_v2.json`）
  - 既存データを元のサイトの最新構造にマッピングする

## 6. 保守性の対応
| ケース | 対応策 |
|--------|--------|
| 元のサイトのHTML構造変更 | バージョン管理でデータを分割保存 |
| 元のサイトの削除 | `/backup`フォルダにデータを移動（削除されても残る） |
| データの不整合 | 24時間以内に自動修正（差分取得のエラーをログに記録） |

## 7. データ保護
| 要件 | 詳細 |
|------|------|
| **SSL通信** | HTTPSで全通信を実装（既定の設定） |
| **データ暗号化** | 重要データ（小説のURL）をAES-256で暗号化（オプション） |
| **アクセス制限** | 本人のみがアクセス可能（認証なしの設定） |

## 8. その他
- **リアルタイム機能**: なし（あなたの要望に沿って設計）
- **テスト環境**: 本番環境とテスト環境を分離（例: `localhost:3000` と `api.tategaki.dev`）
- **メンテナンス**: 月1回の自動チェックでデータの整合性を確認
