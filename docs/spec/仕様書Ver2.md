# **tategaki 仕様書（お気に入り小説を紙の本のように読むサイト）**
## 1. 目的
- 「小説家になろう」「カクヨム」などの小説を、ブラウザ上で**美しい縦書き**で読む。
- お気に入りの更新を自動追跡し、元サイトが削除されてもサーバー上で永続的に読めるようにする。
## 2. ユーザー環境
- **運用**: Dockerコンテナ
- **アクセス**: 基本は誰でも閲覧可能（お気に入り管理機能などは特定ユーザー向け）
- **セキュリティ**: リバースプロキシ側のBasic認証等で制御
## 3. 機能要件
| 機能 | 詳細 |
|------|------|
| **縦書きビューワー** | `writing-mode: vertical-rl` を使用。右から左への横スクロール方式。 |
| **お気に入り管理** | 小説のURLを登録。お気に入りリストの表示。 |
| **自動巡回（Playwright）** | 3時間おきにPlaywrightで元サイトを確認し、新着話をDL。 |
| **削除対応** | 元サイトが404、規約違反、非公開となった場合も、DL済みデータを保持。 |
| **読めなくなった通知** | 元サイトが消失した場合、そのURLを表示し、DL済みの閲覧を促す。 |
## 4. UI/UXデザイン
### (1) カラー・質感
- **背景**: 紙のようなクリーム色（#F5F5DC）や淡いグレー。
- **文字色**: 少し茶色がかった黒（より本に近い色）。
- **フォント**: 「遊明朝」や「Noto Serif JP」などの明朝体フォント。
### (2) ボタン配置 (読書画面)
画面中央または操作パネルに以下の8つのボタンを配置：
1. **次話** | 2. **前話** | 3. **次ページ** | 4. **前ページ** 
5. **目次** | 6. **設定** | 7. **お気に入り** | 8. **ホーム**
## 5. 画面遷移
1. **[ホーム]**: お気に入り一覧を表示。新着があるものはバッジを表示。
2. **[目次]**: 小説選択後に表示（話一覧、元サイトURLへのリンク）。
3. **[読書]**: 本文表示。横スクロールで読み、ボタンで話移動。
4. **[設定]**: フォントサイズ、行間、背景色の切り替え。
## 6. 技術スタック (Technical Stack)
- **Frontend**: HTML5, CSS3, JavaScript (Vanilla JS)
- **Backend**: Node.js (Framework: Express)
- **Crawler**: Playwright (Headless browser)
- **Data storage**: 
    - 作品メタ情報: `data/list.json`
    - 各話本文: `data/novels/[novel_id]/[episode_no].json` (分割保存で高速化)
- **Environment**: Docker / Docker Compose
## 7. 実装の工夫（保守性）
- **データの分割**: 50MBを超えるデータを扱うため、1話1ファイル形式で保存し、ブラウザの負荷を軽減。
- **Playwrightの利用**: HTML構造の変化に強く、JavaScript実行後の内容も確実に取得。
- **ステータス管理**: 「公開中」「元サイト消失」の状態フラグを持ち、消失時は警告と共に保存データを表示。
## 8. セキュリティ・保守
- **SSL**: HTTPS（Proxy側で対応）
- **定期チェック**: 月1回、データの整合性と不整合ログを確認。
