# **tategaki 仕様書 Ver.4 （マルチお気に入りリスト対応）**

## 1. 目的
- 小説をブラウザ上で**美しい縦書き**で読み、更新を自動追跡・永続保存する。
- 複数の「お気に入りリスト（ID管理）」を使い分け、自分好みのライブラリを構築する。

## 2. ユーザー環境と管理
- **運用**: Dockerコンテナ
- **ユーザー管理**: **「お気に入りID」**による仮想的なユーザー分け。
    - 設定画面でIDを入力することで、そのIDに紐付いた小説リストを表示。
    - 各IDごとに「しおり（既読位置）」を管理。

## 3. 機能要件
| 機能 | 詳細 |
|------|------|
| **縦書きビューワー** | `writing-mode: vertical-rl`。右から左へのスクロール。 |
| **マルチリスト管理** | ID別にお気に入りJSONを管理（`data/favs/[id].json`）。 |
| **自動巡回・保存** | 3時間おきにPlaywrightで実行。画像（挿絵）も必要に応じて保存。 |
| **タップ・クリック操作** | 画面両端のタップでページめくり（横スクロール制御）。 |
| **データ永続化** | 元サイト消失時も、DL済み本文・画像をサーバーから表示。 |

## 4. UI/UXデザイン
### (1) 視覚効果
- **フォント**: **游明朝 (Yu Mincho)**。
- **背景/文字色**: クリーム色、淡いグレー、夜間モード（黒背景）などの切り替え。
- **数字**: 2桁は「縦中横」、3桁以上は「縦並び」。

### (2) インタラクション (読書画面)
- **スクロール量**: 画面幅の約90%（1行程度の重なりを残して視線の迷子を防止）。
- **タップエリア**:
    - 右側タップ：次ページ
    - 左側タップ：前ページ
    - 中央タップ：操作メニュー（8つのボタン）の表示/非表示。

## 5. 画面構成と遷移
1. **[ホーム]**: サイト紹介、ID入力フォーム。既にIDがある場合はリストへ。
2. **[目次画面]**: 小説詳細、各話リスト、投稿日時、最新更新情報。
3. **[読書画面]**: 本文、タップ操作によるページ移動。
4. **[設定画面]**: ID切り替え、フォントサイズ、行間、背景色、ボタン表示設定。

## 6. 技術スタック (Technical Stack)
- **Frontend**: HTML5, CSS3, Vanilla JS (タップ検知、スクロール制御)
- **Backend**: Node.js (Express)
- **Crawler**: Playwright (画像ダウンローダー機能付き)
- **Data storage**: 
    - 作品メタ（共通）: `data/novels/[site_type]/[novel_id]/info.json`
    - 本文/画像（共通）: `data/novels/[site_type]/[novel_id]/[ep_no].json`, `data/novels/[site_type]/[novel_id]/images/`
    - リスト/しおり（ID別）: `data/favs/[list_id].json`

## 7. 実装の工夫
- **スクロールスナップ**: CSSの `scroll-snap-type` を使い、ページ移動時に「ピタッ」と止まる心地よさを実現。
- **安全なクローリング**: 複数小説の巡回時、サイトへの負荷を考え1件ごとに3〜5秒の待機時間を挿入。
- **しおり共有**: 同じIDを使えば、PCで読んでいた続きをタブレットで開くことが可能。

## 8. セキュリティ・保守
- **アクセス制御**: Basic認証。
- **ログ**: エラーログの他、「いつ、どのIDで、どの話をDLしたか」の履歴保存。
