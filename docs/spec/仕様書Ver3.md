# **tategaki 仕様書 Ver.3 （お気に入り小説を紙の本のように読むサイト）**

## 1. 目的
- 「小説家になろう」「カクヨム」などの小説を、ブラウザ上で**美しい縦書き**で読む。
- お気に入りの更新を自動追跡し、元サイトが削除されてもサーバー上で永続的に読めるようにする。

## 2. ユーザー環境
- **運用**: Dockerコンテナ
- **アクセス**: 基本は誰でも閲覧可能（お気に入り管理機能などは特定ユーザー向け）
- **セキュリティ**: リバースプロキシ側のBasic認証等で制御

## 3. 機能要件
| 機能 | 詳細 |
|------|------|
| **縦書きビューワー** | `writing-mode: vertical-rl` を使用。右から左への横スクロール方式。 |
| **お気に入り管理** | 小説のURLを登録。お気に入りリストの表示。 |
| **自動巡回（Playwright）** | 3時間おきにPlaywrightで元サイトを確認し、新着話をDL。 |
| **削除対応** | 元サイトが404、規約違反、非公開となった場合も、DL済みデータを保持。 |
| **ステータス表示** | 元サイト消失時、URLを表示し「保存済みデータから表示中」であることを通知。 |

## 4. UI/UXデザイン
### (1) 視覚効果
- **背景**: 紙の質感を意識した色（クリーム色 #F5F5DC / 淡いグレー 等）。
- **文字色**: 本のインクをイメージした深みのある茶色または黒。
- **フォント**: **游明朝 (Yu Mincho)** を基本とする。
- **数字の扱い**: 
    - 2桁: 縦中横（横に並べる）
    - 3桁以上：一文字ずつ縦に並べる（またはフォントの等幅設定で調整）

### (2) 操作パネル (読書画面)
画面中央またはフローティングパネルに以下の8つのボタンを配置：
1. **次話** | 2. **前話** | 3. **次ページ** | 4. **前ページ** 
5. **目次** | 6. **設定** | 7. **お気に入り** | 8. **ホーム**
※設定によりボタン自体の表示/非表示を切り替え可能。

## 5. 画面構成と遷移
1. **[ホーム（入口）]**
    - tategakiの紹介、使い方のまとめ。
    - サイトの「顔」となるランディングページ。
2. **[目次画面]**
    - お気に入りリストや外部（元URL）から遷移。
    - 小説タイトル、作者、章タイトル、各話リストを表示。
    - 投稿日時/更新日時、最新更新日の詳細情報を掲載。
    - 各話タイトルをクリックで「読書画面」へ。
3. **[読書画面]**
    - 本文表示。横スクロールで読み進める。
    - 「次ページ/前ページ」ボタンでスクロール操作。
    - 「次話/前話」でエピソード間移動。
4. **[設定画面]**
    - お気に入りIDの入力/管理。
    - フォントサイズ、行間、背景（紙の色）の調整。
    - UIボタンの表示/非表示設定。

## 6. 技術スタック (Technical Stack)
- **Frontend**: HTML5, CSS3, JavaScript (Vanilla JS)
- **Backend**: Node.js (Framework: Express)
- **Crawler**: Playwright (Headless browser)
- **Data storage**: 
    - 作品管理データ: `data/list.json` 等
    - 本文データ: `data/novels/[novel_id]/[episode_no].json` (1話1ファイル管理)
- **Environment**: Docker / Docker Compose

## 7. 実装の工夫
- **データ管理**: 小説単位のフォルダに各話のJSONを格納し、メンテナンス性を向上。
- **軽量動作**: 読む話のJSONのみを動的に読み込むことで、巨大な小説でも動作を軽く保つ。
- **Playwright活用**: 動的サイトの構造変化に柔軟に対応し、確実なデータ取得を実現。

## 8. 保守・拡張
- **ログ**: 差分取得エラーや元サイト消失をログに記録し、管理者が確認可能にする。
- **将来性**: お気に入りデータのインポート/エクスポート機能の検討。
